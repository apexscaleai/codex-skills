# Web Accessibility Checklist (Next.js)

Mark items PASS/FAIL/UNKNOWN and cite evidence (component/file).

## Semantics

- Headings follow a logical order (PASS/FAIL/UNKNOWN):
- Buttons/links use semantic elements (PASS/FAIL/UNKNOWN):
- Form controls have labels (PASS/FAIL/UNKNOWN):

## Keyboard

- All interactive controls are reachable and usable via keyboard (PASS/FAIL/UNKNOWN):
- Focus is visible and not removed (PASS/FAIL/UNKNOWN):
- Dialogs/menus manage focus correctly (PASS/FAIL/UNKNOWN):

## Images

- Informative images have meaningful `alt` (PASS/FAIL/UNKNOWN):
- Decorative images are ignored by AT (PASS/FAIL/UNKNOWN):

## Errors

- Validation errors are specific and associated with fields (PASS/FAIL/UNKNOWN):
- Error summary/announcement exists where needed (PASS/FAIL/UNKNOWN):

## Visual

- Contrast sufficient for text and key UI elements (PASS/FAIL/UNKNOWN):
- Text scales without breaking layout (PASS/FAIL/UNKNOWN):
